# 비밀번호 찾기
A 회사 총무부서가 분주합니다. 중요한 파일이 모두 들어있는 컴퓨터의 비밀번호를 까먹었기 때문입니다. 
다행히 책상에는 전임자가 놓고간 힌트가 담긴 쪽지가 놓여있습니다. 
쪽지의 내용은 이러합니다.

'비밀번호는 쉴새없이 변하며, 컴퓨터 화면에는 매 번 일련의 문자들이 주어집니다. 
단 컴퓨터 화면에서 알파벳 c까지의 알파벳 문자열이 비밀번호입니다.'

머리가 좋은 김대리는 바로 이해를 하고 암호를 풀었습니다. 
이때 김대리가 사용한 방법을 프로그램으로 작성하세요.

'''java
public class Main {
    public static void main(String[] args) throws IOException { 

        Scanner scanner = new Scanner(System.in); 
        
        for (int i = 0; i<20; i++) {
            char c = scanner.next().charAt(0);
            System.out.println(c);
            
            if (c == 'c') {
                break;
            }
        }
    }
}
코드풀이 및 설명:
이 프로그램은 문자를 하나씩 입력받아 바로 출력하며, 입력된 문자가 알파벳 'c'일 때까지 반복한다.
문제를 통해 비밀번호는 **컴퓨터 화면에 나타난 문자들 중 'c'가 등장할 때까지의 문자열**임을 알 수 있으므로, 입력된 문자를 출력하다가 'c'가 입력되면 반복을 종료한다.
이를 위해 for문으로 최대 20번까지 입력을 받도록 설정하고, 각 입력마다 문자를 출력한 뒤 'c'인지 확인하여 조건을 만족하면 즉시 종료하도록 구현하였다.
최종적으로 출력된 문자들의 순서가 곧 비밀번호가 된다.
'''

#삼각형의 조건
수학을 좋아하는 진수는 삼각형을 가지고 놀던 중 다음과 같은 생각이 들었습니다.
평면과 공간에서 삼각형의 내각의 합은 180도이고 각각의 각의 크기가 0도 보다 커야하잖아. 그렇다면 N차원 공간의 삼각형도 그러하지 않을까?
진수는 세 각의 합이 180도이고 각각의 각이 0도보다 큰 삼각형을 '버뮤다 삼각형'이라고 부르기로 했습니다.
세 각의 크기가 주어질 때, 버뮤다 삼각형이 될 수 있는지 없는지 판단하는 프로그램을 만드세요.

'''java
public class Main {
    public static void main(String[] args) throws IOException { 

        Scanner scanner = new Scanner(System.in); 

        int a = scanner.nextInt(); 

        int b = scanner.nextInt();
        
        int c = scanner.nextInt();
        
        if (a+b+c==180 && a > 0 && b > 0 && c > 0) {
            System.out.println('P'); 
        } else {
            System.out.println('F');
        }
    }
}
'''

코드풀이 및 설명:
이 프로그램은 세 개의 각의 크기를 입력받아 버뮤다 삼각형이 될 수 있는지 판단한다.
버뮤다 삼각형의 조건은 세 각의 합이 180도이고 각각의 각이 0보다 커야 한다는 것이므로, 입력받은 세 정수 a, b, c에 대해 이 조건을 if문으로 확인한다.
조건을 모두 만족하면 버뮤다 삼각형이 가능하므로 'P'를 출력하고, 하나라도 만족하지 않으면 'F'를 출력하도록 구현하였다.

#소수구하기
소수 정리'란 어떤 자연수 이하의 소수가 몇 개나 있는지 그 값을 어림할 수 있는 정리입니다.
이 정리가 맞는지 의구심이 들은 진수는 직접 확인해 보기로 했습니다.
한참을 계산하던 진수는 소수의 개수를 헤아리는데 몹시 많은 시간이 든다는 걸 깨닫고 여러분들께 도움을 구했습니다.
진수를 위해 자연수 N 이하의 소수 개수를 출력하는 프로그램을 만들어주세요.
이때 소수란 양의 약수가 자기 자신과 1만 존재하는 수를 말합니다.
또한 어떤 정수 A가 어떤 정수 B의 약수라는 것은 B를 A로 나누었을 때 나누어떨어짐을 의미합니다.

'''java
public class Main {
    public static void main(String[] args) throws IOException { 

        Scanner scanner = new Scanner(System.in); 
        
        int N = scanner.nextInt();

        boolean[] isPrime = new boolean[N + 1];

        // 2 이상은 일단 소수로 가정
        for (int i = 2; i <= N; i++) {
            isPrime[i] = true;
        }

        // 에라토스테네스의 체
        for (int i = 2; i * i <= N; i++) {
            if (isPrime[i]) {
                for (int j = i * i; j <= N; j += i) {
                    isPrime[j] = false;
                }
            }
        }

        // 소수 개수 세기
        int count = 0;
        for (int i = 2; i <= N; i++) {
            if (isPrime[i]) {
                count++;
            }
        }

        System.out.println(count);


    }
}
'''

코드풀이 및 설명:
이 프로그램은 자연수 N 이하에 존재하는 소수의 개수를 구한다.
먼저 isPrime 배열을 사용하여 2 이상인 수를 모두 소수로 가정한 뒤, 에라토스테네스의 체 알고리즘을 이용해 소수가 아닌 수들을 제거한다.
구체적으로 어떤 수 i가 소수라면, i의 배수들은 소수가 아니므로 false로 표시한다.
이 과정을 거친 후 isPrime 배열에서 true로 남아 있는 수들의 개수를 세어 출력함으로써 N 이하의 소수 개수를 효율적으로 계산한다.

#아까운 쿠폰
피자집 사장인 현석이는 대한민국 화폐 단위와 같은 쿠폰을 만들어 손님들이 현금으로 결제할 때 거스름돈 대신 쿠폰으로 거슬러 줍니다.
하지만 최근 쿠폰이 모자라진 현석이는 손님들에게 거슬러 주는 쿠폰의 개수를 가능한 적게 하려고 합니다.

현석이가 거슬러주어야 하는 금액이 주어지면, 그 금액에 맞추어 거슬러줄 때 쿠폰이 최소 몇 장 필요한지 출력하는 프로그램을 작성하세요.
(쿠폰은 대한민국 화폐 단위와 동일하게 10원, 50원, 100원, 500원, 1000원, 5000원, 10000원, 50000원짜리가 있습니다.)

'''java
public class Main {
    public static void main(String[] args) throws IOException { 

        Scanner scanner = new Scanner(System.in); 

        int money = scanner.nextInt();
        
        int[] coupon = {50000, 10000, 5000, 1000, 500, 100, 50, 10};
        
        int count = 0;
        
        for (int i=0; i<coupon.length; i++) {   //탐욕적 알고리즘(O(1))
            count = count + (money / coupon[i]);
            money = money % coupon[i];
        }

        System.out.println(count); 

    }
}
'''

코드풀이 및 설명:이 프로그램은 거슬러주어야 할 금액을 입력받아 필요한 쿠폰의 최소 개수를 구한다.
대한민국 화폐 단위와 동일한 쿠폰이 큰 금액부터 준비되어 있으므로, 가장 큰 단위의 쿠폰부터 가능한 만큼 사용하는 탐욕적 알고리즘을 적용한다.
각 쿠폰 단위에 대해 현재 금액을 나눈 몫만큼 쿠폰 개수를 더하고, 나머지 금액으로 갱신하는 과정을 반복한다.
이 방법을 통해 전체 쿠폰 개수를 최소로 하여 효율적으로 거스름돈을 계산할 수 있다.
